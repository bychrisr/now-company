## Story

**As a** membro de uma empresa,
**I want** uma interface em `/company/settings/social-accounts` para conectar, listar, desconectar e disparar sync manual das contas sociais da minha empresa,
**so that** eu gerencie a presença social da empresa sem precisar de acesso ao banco ou API.

## Acceptance Criteria

1. Existe nova página `ui/src/pages/CompanySocialAccounts.tsx` (ou nome equivalente).
2. Rota `/company/settings/social-accounts` registrada e protegida (apenas membros da empresa).
3. Página lista contas conectadas em formato de cards/tabela com:
   - Ícone da plataforma + nome
   - Handle (ex: @paperclipai)
   - Follower count + engagement rate
   - `last_synced_at` (ex: "Sincronizado há 2h")
   - Status visual: ativo / `needs_reauth` (com badge de alerta)
   - Ações: "Sincronizar agora", "Desconectar"
4. Seção "Conectar nova conta": lista plataformas disponíveis (de `social_platforms` com `status=enabled`) que a empresa AINDA NÃO conectou. Botão "Conectar" inicia OAuth flow (chama endpoint da Story 1.5).
5. Callback de OAuth (`?status=connected&platform=...`) exibe toast/notification de sucesso ao retornar para a página.
6. "Sincronizar agora" chama `POST /api/company/social-accounts/:id/sync` e atualiza UI com novos valores.
7. "Desconectar" pede confirmação (modal) antes de chamar DELETE.
8. Estados de loading/erro tratados (skeleton durante fetch, toast em erro).
9. Item "Redes Sociais" adicionado ao sidebar de empresa (verificar componente equivalente a `InstanceSidebar` mas para escopo de empresa).
10. UI segue padrão visual existente (consultar páginas em `ui/src/pages/` que tratam de settings de empresa).
11. Responsivo (mobile-first per CLAUDE.md).
12. `pnpm run typecheck`, `pnpm run build`, `pnpm run test` passam.
