## Story

**As a** Super Admin,
**I want** o seed populado com 15+ plataformas e seus specs técnicos completos (capabilities, copy_specs, image_specs),
**so that** o catálogo global esteja pronto para uso pelas empresas e os agentes de geração de conteúdo tenham referência técnica precisa de cada plataforma (formatos, dimensões, limites de copy).

## Acceptance Criteria

1. Existe arquivo TypeScript com types fortes para os JSONBs:
   - `PlatformCapabilities` (ex: `supports_video, supports_carousel, supports_stories, supports_dm, supports_live, supports_ads`)
   - `PlatformCopySpecs` (ex: `max_title_chars, max_body_chars, max_hashtags, supports_mentions, supports_emoji`)
   - `PlatformImageSpecs` (ex: arrays de `{ slot, width, height, aspect_ratio, format }` para feed/story/thumbnail/etc.)
2. Types localizados em `packages/shared/src/types/social-platforms.ts` (ou path equivalente seguindo padrão do projeto).
3. Schema Drizzle de `social_platforms.ts` (criado na Story 1.1) atualizado para usar esses types em `capabilities/copy_specs/image_specs` via `.$type<...>()`.
4. Seed expandido com no mínimo 15 plataformas do atlas (`docs/references/global-content-formats-atlas.md` v3.1):
   instagram, whatsapp-business, facebook, threads, youtube, tiktok, kwai, twitch, linkedin, discord, reddit, hacker-news, indie-hackers, quora, product-hunt, blog, substack, github, twitter-x, bluesky, pinterest, telegram, spotify, apple-podcasts, google-business
5. Cada plataforma tem `capabilities`, `copy_specs`, `image_specs` preenchidos conforme dados do atlas (não inventar — usar números reais documentados).
6. Seed continua idempotente — rodar 2x não duplica nem corrompe dados existentes (usar upsert/`onConflictDoUpdate` em `slug`).
7. Plataformas excluídas (Koo, Nimo TV, Rumble, Band.us) NÃO entram no seed.
8. `pnpm run typecheck`, `pnpm run build`, `pnpm run db:seed` passam.

### Padrão de upsert idempotente (Drizzle)
```typescript
await db.insert(socialPlatforms)
  .values(platforms)
  .onConflictDoUpdate({
    target: socialPlatforms.slug,
    set: {
      name: sql`excluded.name`,
      category: sql`excluded.category`,
      capabilities: sql`excluded.capabilities`,
      copySpecs: sql`excluded.copy_specs`,
      imageSpecs: sql`excluded.image_specs`,
      updatedAt: new Date(),
    },
  });
```
