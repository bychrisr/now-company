### Schema proposto da tabela `social_platforms`

```typescript
// packages/db/src/schema/social_platforms.ts
import { pgTable, uuid, text, timestamp, jsonb, index, uniqueIndex } from "drizzle-orm/pg-core";

export const socialPlatforms = pgTable(
  "social_platforms",
  {
    id: uuid("id").primaryKey().defaultRandom(),

    // Identificação
    slug: text("slug").notNull(), // 'instagram', 'youtube' — único global
    name: text("name").notNull(), // 'Instagram', 'YouTube' — display
    category: text("category").notNull(), // 'social', 'video', 'audio', 'blog', 'community'

    // Governança Super Admin
    status: text("status").notNull().default("enabled"), // enabled | disabled
    sortOrder: integer("sort_order").notNull().default(0),

    // Specs técnicos (expandido na Story 1.3)
    capabilities: jsonb("capabilities").$type<Record<string, unknown>>().notNull().default({}),
    copySpecs: jsonb("copy_specs").$type<Record<string, unknown>>().notNull().default({}),
    imageSpecs: jsonb("image_specs").$type<Record<string, unknown>>().notNull().default({}),

    // Metadata
    iconUrl: text("icon_url"),
    description: text("description"),
    websiteUrl: text("website_url"),

    createdAt: timestamp("created_at", { withTimezone: true }).notNull().defaultNow(),
    updatedAt: timestamp("updated_at", { withTimezone: true }).notNull().defaultNow(),
  },
  (table) => ({
    slugUq: uniqueIndex("social_platforms_slug_uq").on(table.slug),
    statusSortIdx: index("social_platforms_status_sort_idx").on(table.status, table.sortOrder),
    categoryIdx: index("social_platforms_category_idx").on(table.category),
  }),
);
```

**Justificativas (porquê):**
- `slug` é PK lógica para uso em código (`platform_id` em FKs futuras). UUID `id` é PK física para joins eficientes.
- `capabilities/copy_specs/image_specs` são JSONB porque cada plataforma tem estrutura própria — schema rígido seria contraproducente. Story 1.3 define os shapes específicos via TypeScript.
- Sem `company_id`: tabela GLOBAL gerida pelo Super Admin (Bloco 0 → Story 1.2).
- Sem soft delete nesta story — `status: 'disabled'` cobre o caso de uso.

### Padrões a seguir

- **Schema:** seguir formato de `packages/db/src/schema/companies.ts` (imports, export, índices nomeados com pattern `{table}_{cols}_{type}`).
- **Migration:** Drizzle Kit gera automaticamente — não editar manualmente o `.sql` exceto se necessário (e documentar o porquê).
- **Seed:** seguir padrão de `packages/db/src/seed.ts` existente (usa `db.insert(...).values(...)`).
