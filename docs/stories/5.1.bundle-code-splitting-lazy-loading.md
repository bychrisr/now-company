## Story

**As a** usuário do Paperclip,
**I want** que as páginas carreguem mais rapidamente,
**so that** eu tenha uma experiência fluída mesmo quando navego para seções complexas como Detalhes do Agente e Chat de Issues.

1. `AgentDetail.tsx` é convertido para carregamento via `React.lazy()` + `Suspense` nas suas 3 rotas em `App.tsx` (linhas 98–100: `agents/:agentId`, `agents/:agentId/:tab`, `agents/:agentId/runs/:runId`)
2. `IssueChatThread.tsx` (componente em `ui/src/components/`) é convertido para `React.lazy()` + `Suspense` no ponto de consumo em `IssueDetail.tsx`
3. O Suspense boundary de `AgentDetail` usa `<PageSkeleton variant="agent-detail" />` — nova variante adicionada ao componente existente `ui/src/components/PageSkeleton.tsx`
4. O Suspense boundary de `IssueChatThread` usa `<PageSkeleton variant="issue-chat" />` — nova variante adicionada ao componente existente `ui/src/components/PageSkeleton.tsx`
5. Os skeletons das duas variantes novas mantêm o layout da página (sem flash branco/vazio) — estrutura visual correspondente ao layout real do componente
6. `pnpm build` produz análise de bundle via `rollup-plugin-visualizer` (ou equivalente já existente) que demonstra redução de ≥ 40% no chunk principal em relação ao baseline pré-task
7. `pnpm build` completa sem warnings de bundle size

9. Navegação para as páginas refatoradas (`/agents/:id`, `/agents/:id/:tab`, `/agents/:id/runs/:runId`, `IssueDetail` com thread) funciona identicamente ao comportamento anterior
10. LCP (Largest Contentful Paint) não aumenta mais que 200ms medido via Lighthouse em relação ao baseline documentado antes desta task

## Dev Notes

### Contexto da Story

Story implementa **Bloco A — Bundle Size Critical** do EPIC-005. Os dois maiores componentes do bundle foram identificados no Discovery ORION:

- `AgentDetail.tsx` — **176kb** (não minificado), em `ui/src/pages/AgentDetail.tsx`
- `IssueChatThread.tsx` — **158kb** (não minificado), em `ui/src/components/IssueChatThread.tsx`

Juntos, esses dois componentes representam uma parcela significativa do bundle inicial, causando TTI (Time to Interactive) elevado mesmo em navegações que não precisam desses componentes.

### Paths confirmados (verificados via grep e listagem de diretório)

```
ui/src/App.tsx                              # imports de AgentDetail na linha 11; rotas nas linhas 98–100
ui/src/pages/AgentDetail.tsx                # componente alvo — 176kb
ui/src/components/IssueChatThread.tsx       # componente alvo — 158kb
ui/src/pages/IssueDetail.tsx               # consumidor de IssueChatThread (confirmar via grep)
ui/src/components/PageSkeleton.tsx          # skeleton existente — adicionar variantes "agent-detail" e "issue-chat"
```

### Estrutura atual de rotas do AgentDetail em App.tsx

```tsx
// Linhas 98–100 de ui/src/App.tsx (verificado):
<Route path="agents/:agentId" element={<AgentDetail />} />
<Route path="agents/:agentId/:tab" element={<AgentDetail />} />
<Route path="agents/:agentId/runs/:runId" element={<AgentDetail />} />
```

### Padrão de named export com React.lazy

O projeto usa named exports (`export function AgentDetail`), não default exports. O padrão correto de conversão é:

```tsx
// Named export → lazy loading:
const AgentDetail = React.lazy(() =>
  import("./pages/AgentDetail").then((m) => ({ default: m.AgentDetail }))
);
```

### Atenção: IssueChatThread não tem rota própria

`IssueChatThread` é um componente (não uma página) consumido internamente por `IssueDetail.tsx`. O lazy loading deve ser aplicado **no ponto de importação em `IssueDetail.tsx`**, não no `App.tsx`. O Suspense boundary deve ser posicionado de forma a não quebrar o layout existente de `IssueDetail`.

### PageSkeleton.tsx já existe — NÃO recriar

O componente `PageSkeleton.tsx` em `ui/src/components/` já existe com variantes: `list`, `issues-list`, `detail`, `dashboard`, `approvals`, `costs`, `inbox`, `org-chart`. Esta story **adiciona** apenas as variantes `"agent-detail"` e `"issue-chat"` ao arquivo existente. NÃO criar `PageSkeleton` do zero.

### Padrões a seguir

- **Skeleton:** Usar exclusivamente `<Skeleton>` de `@/components/ui/skeleton` com classes Tailwind — sem CSS inline, sem hardcode de cores
- **Design system:** Tokens OKLCH definidos em `ui/src/index.css` (`:root`) — ex: `border-border`, `bg-muted`
- **Testes:** Vitest + Testing Library — padrão já estabelecido no projeto

### Decisões fixas (não eliciar)

- **Escopo restrito:** Esta story aplica lazy loading APENAS em `AgentDetail` e `IssueChatThread`. Não extender para outros componentes nesta iteração.
- **Sem SSR:** Projeto é CSR puro (Vite + React) — não há risco de quebra de SSR com `React.lazy`.
- **Rollback trivial:** Reverter para import direto é uma operação de 1 linha por componente.

### Out of scope (vai em outras stories)

- ❌ Lazy loading de outros componentes → backlog EPIC-005 (fora das 3 stories deste epic)
- ❌ Prefetch de rotas → pode ser avaliado em iteração futura após validação dos ganhos desta story
- ❌ Service Worker / cache de assets → ORION-UI Sprint 2
- ❌ Componentes de estado loading/error/empty → Story 5.2
