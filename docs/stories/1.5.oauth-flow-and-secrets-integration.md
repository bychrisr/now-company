## Story

**As a** empresa do Paperclip,
**I want** conectar minhas contas sociais via OAuth (a partir do Instagram como primeira plataforma),
**so that** os tokens fiquem isolados no cofre `company_secrets` e a tabela `company_social_accounts` apenas referencie via `secret_id`.

## Acceptance Criteria

1. Existe endpoint `POST /api/company/social-accounts/connect/:platformSlug` que inicia OAuth flow:
   - Gera state token (para CSRF)
   - Redireciona para URL de autorização da plataforma
   - Valida permissão de quem chama (membro da empresa)
2. Existe callback `GET /api/oauth/callback/:platformSlug?code=...&state=...`:
   - Valida state token
   - Troca `code` por `access_token` + `refresh_token`
   - Cria registro em `company_secrets` (key: `oauth_{platformSlug}_{platformAccountId}`)
   - Cria/atualiza registro em `company_social_accounts` com `secret_id` apontando para o secret recém-criado
   - Redireciona para UI de sucesso
3. Endpoint `DELETE /api/company/social-accounts/:id`:
   - Soft delete via `is_active = false`
   - Marca secret como `status = 'revoked'` em `company_secrets` (NÃO deletar — mantém audit trail)
4. Endpoint `GET /api/company/social-accounts` — lista contas da empresa logada (com `is_active = true` por padrão).
5. **Implementar fluxo completo APENAS para Instagram** nesta story. Outras plataformas viram stories próprias quando necessário (1.5.x).
6. Isolamento: usuário da empresa A não consegue listar/conectar/desconectar contas da empresa B.
7. Tokens criptografados via mecanismo existente do `company_secrets` (sem reimplementar criptografia).
8. `pnpm run typecheck`, `pnpm run build`, `pnpm run test` passam.
